# ADR-001 하이브리드 저장 전략 채택

상태: Accepted  
작성일: 2026-03-18

---

## Context

황총무는 단일 제품처럼 보이지만 실제로는 서로 다른 도메인과 사용 패턴을 가진 도구들의 집합이다.  
모든 메뉴를 Supabase로 통일하면 초기 구현과 유지 비용이 커지고, 모든 메뉴를 localStorage로 두면 공유/방 기반 기능이 불가능하다.

---

## Decision

도메인 특성에 따라 저장소를 분리한다.

- 공유 / 방 / 협업 / 서버 저장이 필요한 메뉴는 Supabase 사용
- 개인용 / 빠른 프로토타이핑 / 단일 브라우저 사용이 허용되는 메뉴는 localStorage 사용

---

## Rationale

- 제품 개발 속도를 높일 수 있다
- 초기 설계 비용을 줄일 수 있다
- 기능별 복잡도를 필요한 만큼만 지불할 수 있다

---

## Consequences

### Positive

- 빠르게 다양한 메뉴를 출시 가능
- 서버 비용/설계 비용을 기능별로 선택 가능
- 개인 도구와 협업 도구를 같은 앱 안에 수용 가능

### Negative

- 저장 전략 일관성이 깨진다
- 백업/복구 정책이 메뉴마다 다르다
- 사용자 기대치가 달라질 수 있다

---

## Follow-up

- localStorage 메뉴 export/import 제공
- 저장 전략 문서화
- 장기적으로 계정 기반 통합 여부 재검토
