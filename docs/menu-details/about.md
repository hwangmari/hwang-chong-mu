# 소개/문의 상세 분석

## 1. 메뉴 목적

사이트 소개, 제공 중인 서비스 목록, 기술 스택, 문의처를 한 페이지에 보여주는 정보 페이지다.
Google AdSense 승인에 필요한 "About" 페이지 역할을 한다.

## 2. 라우트 구조

- 소개 페이지: `/about`

## 3. 핵심 파일 구조

```text
app/about/page.tsx         # 소개/문의 페이지
```

단일 파일로 구성. 별도 컴포넌트 분리 없음.

## 4. 페이지 구조

`PageIntro` + 4개의 `StSection`으로 구성된다.

### 섹션 구성

1. **이 사이트는요** — 황총무의 실험실 소개, 제작 배경, 제작자 소개
2. **제공 중인 서비스** — 10개 서비스 그리드 (이모지 + 이름 + 설명)
3. **기술 스택** — Frontend, Styling, Backend, Infra 4개 항목
4. **문의하기** — Email, GitHub, Blog 링크

### 하단 링크

- 개인정보처리방침(`/privacy`) 페이지로의 내부 링크

## 5. 저장 방식

정적 페이지. DB 연동 없음. 서비스 목록은 컴포넌트 내 상수 배열로 관리.

## 6. 기술적 강점

- 기존 공통 컴포넌트(PageIntro, StContainer, StWrapper, StSection) 재사용
- 서비스 그리드 반응형 (2열 → 모바일 1열)
- 외부 링크(`mailto:`, GitHub, Blog)와 내부 링크(`/privacy`) 적절히 혼합

## 7. 구조적 한계

- 서비스 목록이 하드코딩되어 새 서비스 추가 시 수동 수정 필요
- 문의 폼 없음 (이메일 링크만 제공)

## 8. 개선 제안

- 서비스 목록을 ROUTE_CONFIG(GlobalHeader)와 연동하여 자동 동기화
- 문의 폼 추가 (Supabase에 저장 또는 이메일 전송)
- FAQ 섹션 추가
