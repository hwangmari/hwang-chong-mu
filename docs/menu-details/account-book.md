# 가계부 상세 분석

## 0. 포트폴리오용 요약

가계부는 단순한 수입/지출 입력 화면이 아니라, 개인 기록과 공용 정산을 함께 다루는 생활형 재무 기록 도구다.

핵심 포인트:

- 개인 가계부에서 먼저 원본 내역을 정리하고
- 필요한 항목만 공용 가계부방으로 연결하며
- 같은 데이터를 리스트 / 캘린더 / 보드 관점으로 다시 확인할 수 있다

즉, "혼자 기록하는 흐름"과 "함께 보는 흐름"을 하나의 서비스 안에서 자연스럽게 이어보려는 프로젝트다.

## 1. 메뉴 목적

가계부 메뉴는 단순한 개인 장부가 아니라 "개인 가계부 + 공용 가계부방 + 공유/미러링" 구조를 가진 재무 기록 시스템이다.  
최근에는 자연어 입력 중심 UX로 단순화가 진행되고 있다.

## 2. 라우트 구조

- 허브: `/account-book`
- 연간 상세: `/account-book/annual`
- 워크스페이스 선택: `/account-book?workspaceId=...`

## 3. 핵심 파일 구조


## 4. 저장 방식 분석

현재 가계부는 Supabase RPC를 중심으로 저장하고, 브라우저 로컬 데이터는 레거시 마이그레이션 및 일부 접근 상태 보조 용도로 남아 있다.

스토어는 RPC 응답을 기준으로 정규화된다.

관련 저장/정규화 축:

- `ACCOUNT_BOOK_STORE_KEY`
- `LEGACY_ACCOUNT_BOOK_KEY`
- `account_book_get_store`
- `account_book_upsert_user`
- `account_book_upsert_workspace`
- `account_book_upsert_entry`
- `account_book_toggle_share_link`

스토어 구조:

- `users`
- `workspaces`
- `entries`
- `shareLinks`

즉, 지금 구조는 "브라우저 안 임시 장부"보다는 작은 멀티 워크스페이스 가계부 시스템에 더 가깝다.

## 5. 데이터 모델 분석

### 사용자

- 이름
- 비밀번호
- 개인 워크스페이스 ID

### 워크스페이스

- 개인/공용 타입
- 이름
- 비밀번호
- 소유자
- 멤버 IDs

### 엔트리

- 날짜
- 작성자
- 수입/지출
- 카테고리/세부 카테고리
- 가맹점
- 항목
- 금액
- 결제 수단
- 메모
- 자연어 원문

### 공유 링크

- 어떤 엔트리가
- 어느 워크스페이스에서
- 어느 워크스페이스로
- 누구에 의해 공유되었는가

## 6. 화면 구조

### 허브

`WorkspaceHub`는 개인 가계부와 공용 가계부방을 나눠 보여준다.

역할:

- 워크스페이스 목록 진입
- 설정 모달 진입

### 상세

`WorkspaceLedgerView`는 사실상 가계부 실제 사용 화면이다.

포함 기능:

- 자연어 입력
- 월별 리스트 보기
- 캘린더 보기
- 보드 보기
- 참여자 필터
- 월 요약
- 직접 입력 모달
- 공유 토글

### 설정

`WorkspaceSettingsModal`은 사용자/공용방 구조를 편집하는 관리자 화면이다.

## 7. 자연어 입력 UX 분석

최근 가계부는 "입력 복잡도"를 줄이기 위해 자연어 파서를 포함한다.

예:

- `3월 17일 네이버쇼핑 마라샹궈 구매 16만원`
- `오늘 스타벅스 라떼 카드 6100원`

해석 대상:

- 날짜
- 가맹점
- 항목
- 결제수단
- 금액
- 카테고리

한글 단위 금액도 처리하도록 확장된 점이 특징이다.

이 방향은 이 메뉴에서 가장 중요한 UX 개선 축이다.

## 8. 조회/합성 구조

`resolveWorkspaceEntries`는 직접 엔트리만 보여주는 것이 아니라 다음을 합친다.

- 직접 작성 엔트리
- 공유 링크로 들어온 엔트리
- 공용방에서 개인방으로 미러링된 엔트리

즉, 이 메뉴는 "원본 데이터"와 "보여지는 데이터"가 다를 수 있다.

이 구조의 장점:

- 공용방/개인방 역할을 동시에 만족시킨다

이 구조의 비용:

- 학습 난도가 올라간다
- 디버깅 난도가 높다

## 9. 사용자 시나리오

### 개인 기록 시나리오

1. 개인 가계부방 입장
2. 자연어 또는 수동 입력
3. 월별 리스트/캘린더/보드 확인
4. 필요 시 공용방 공유

### 가정/공용 시나리오

1. 공용 가계부방 입장
2. 멤버별/전체 보기 전환
3. 월별 소비 비교
4. 연간 상세 확인

## 10. 기술적 강점

- 개인/공용 회계 구조가 분리되어 확장성이 높다
- Supabase RPC + 스토어 정규화 구조로 데이터 흐름이 비교적 일관적이다
- 레거시 로컬 데이터 마이그레이션을 고려하고 있다
- 자연어 입력, 이미지 등록, 직접 입력을 함께 제공해 입력 진입장벽을 낮춘다
- 리스트 / 캘린더 / 보드로 같은 데이터를 다른 관점에서 읽게 해준다

## 11. 한계

- 개인/공용/공유 개념이 풍부한 만큼 첫 진입 학습 비용이 있다
- 모바일에서 헤더, 모달, 상세 패널 밀도가 조금만 어긋나도 사용 피로가 커진다
- 협업 구조가 들어가면서 단순 가계부보다 상태와 예외 케이스가 많다

## 12. 개선 제안

- export/import 기능 추가
- 개인방 로그인/재진입 흐름을 더 명확한 문구와 피드백으로 고도화
- 모바일 레이아웃과 팝오버/모달 접근성 계속 정리
- 자연어 입력 정확도 향상
- 워크스페이스 설정 UX 단순화
- 개인/공용 가계부의 역할 설명을 화면에서 더 친절하게 제공
