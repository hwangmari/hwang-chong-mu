# 일일 기록 상세 분석

## 1. 메뉴 목적

일일 기록은 "한 줄 일기 + 체크리스트"를 월 단위 기록장으로 관리하는 개인용 라이프로그 서비스다.

## 2. 라우트 구조

- 생성: `/daily`
- 상세: `/daily/[id]`

## 3. 핵심 파일 구조

```text
app/daily/page.tsx
app/daily/[id]/page.tsx
app/daily/storage.ts
```

파일 수는 적지만, storage 추상화와 상세 페이지 로직의 밀도가 높다.

## 4. 저장 방식 분석

이 메뉴는 localStorage 기반이다.

저장 데이터:

- 기록장 설정
  - 제목
  - 기본 체크리스트
  - 월별 체크리스트
  - 비밀번호
  - 테마 컬러
- 월별 날짜 엔트리
  - 일기
  - 체크 배열

이 구조는 "서버 없이도 꽤 완성도 높은 개인 기록장"을 제공하기 위한 설계다.

## 5. 생성 페이지 분석

생성 단계에서 바로 제품 핵심 모델이 잡힌다.

- 기록장 제목
- 개인 비밀번호
- 테마 컬러
- 체크리스트 항목
- 기존 기록장 바로가기

즉, 단순 생성 페이지이지만 기록장 관리 홈 역할도 함께 한다.

## 6. 상세 페이지 분석

상세 페이지는 localStorage형 메뉴 중 가장 구조가 탄탄하다.

주요 기능:

- 월 이동
- 월별 체크리스트 수정
- 날짜별 일기 입력
- 체크 토글
- 평균 점수 계산
- trend graph 생성
- 비밀번호 설정/변경/해제
- 테마 컬러 변경

상태도 상당히 세밀하다.

- access modal
- access notice
- 현재 월
- 월별 draft checklist
- version 기반 리렌더

## 7. storage.ts 분석

`storage.ts`의 장점은 추상화 수준이 좋다는 점이다.

주요 기능:

- 날짜 키 생성
- 월 키 생성
- 기록장 정규화
- 월별 체크리스트 조회/저장
- 월별 날짜 엔트리 생성/정렬/저장

즉, UI가 storage 구현 세부사항을 직접 몰라도 되도록 잘 감쌌다.

## 8. 사용자 시나리오

1. 기록장 생성
2. 월별 체크리스트 구성
3. 날짜별 일기와 체크 기록
4. 월간 점수와 추세 확인
5. 필요 시 비밀번호/컬러 변경

## 9. 기술적 강점

- localStorage 기반 치고 모델 설계가 매우 좋다
- 월별 체크리스트를 따로 관리하는 점이 실사용에 강하다
- 잠금 해제 상태를 sessionStorage로 분리해 UX를 세심하게 설계했다
- trend graph, 평균 점수 등 회고 포인트가 잘 살아 있다

## 10. 한계

- 클라우드 백업이 전혀 없다
- 여러 기기에서 같은 기록장을 쓰기 어렵다
- 협업/공유 기능은 거의 없다

## 11. 개선 제안

- export/import
- PDF 또는 이미지 내보내기
- 월별 회고 템플릿 자동 생성
- 주간 보기
- 기록장 복제 기능
