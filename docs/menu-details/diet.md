# 체중 관리 상세 분석

## 1. 메뉴 목적

체중 관리는 몸무게와 식단을 함께 기록하면서 주간/월간 변화를 추적하는 건강 로그 도구다.

## 2. 라우트 구조

- 생성: `/diet`
- 상세: `/diet/[id]`

## 3. 핵심 파일 구조

```text
app/diet/page.tsx
app/diet/[id]/page.tsx
app/diet/DietMainContent.tsx
app/diet/DietMealInput.tsx
app/diet/WeightChart.tsx
app/diet/AnalysisCard.tsx
```

## 4. 생성 페이지 분석

입력값:

- 목표 이름
- 목표 몸무게

구조는 단순하지만 목적이 명확하다.  
사용자는 목표만 정하면 곧바로 기록 화면으로 이동한다.

## 5. 상세 화면 분석

상세 페이지는 `DietMainContent`에 거의 모든 로직이 모여 있다.

핵심 상태:

- 현재 날짜
- 주간/월간 보기 모드
- 오늘 로그
- 초기 로그
- 어제 로그
- 차트 로그
- 저장 중 상태

즉, 이 메뉴는 "하루 입력 + 기간 분석"이 동시에 들어있는 구조다.

## 6. 주요 기능

- 아침/점심/저녁 식단 입력
- 아침/점심/저녁 몸무게 입력
- 메모 입력
- 주간/월간 차트
- 전날 대비 변화량 계산
- 저장되지 않은 변경 감지

특히 `isDirty`와 `beforeunload` 처리로 기록 중 이탈 방지를 고려한 점이 좋다.

## 7. DB 구조 추정

### `diet_goals`

- 목표 이름
- 목표 몸무게

### `diet_logs`

- goal_id
- date
- 식사 필드
- 시간대별 몸무게 필드
- memo

모델은 직관적이며 확장도 쉽다.

## 8. 사용자 시나리오

1. 다이어트 목표 생성
2. 특정 날짜 기록 조회
3. 식단/몸무게 입력
4. 저장
5. 주간/월간 변화 확인
6. 전날 대비 분석 확인

## 9. 기술적 강점

- 입력과 분석이 한 화면에서 자연스럽다
- 주/월 전환 구조가 명확하다
- 식단과 몸무게를 따로 보지 않고 함께 본다
- 실사용성 높은 dirty check가 들어 있다

## 10. 한계

- 운동, 수면, 컨디션 같은 주변 지표가 없다
- 식단은 문자열 기반이라 구조화 분석이 약하다
- 공유/코칭/회고 기능은 아직 없다

## 11. 개선 제안

- 운동량/수면/컨디션 필드 추가
- 목표 달성률 카드
- 식단 태그 자동 분석
- 주간 요약 리포트
- 목표 체중까지 남은 추세 예측
