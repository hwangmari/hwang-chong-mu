# 게임방 상세 분석

## 1. 메뉴 목적

게임방은 친구들과 가볍게 즐길 수 있는 복불복/즉석 게임 허브다.  
로컬 즉시 실행형과 실시간 방 참여형이 동시에 존재하는 것이 가장 큰 특징이다.

## 2. 라우트 구조

- 로비: `/game`
- 방 게임: `/game/[id]`
- 빠른 시작: `/game/quick/[id]`

## 3. 핵심 파일 구조

```text
app/game/page.tsx
app/game/[id]/page.tsx
hooks/useGameRoom.ts
app/game/quick/[id]/page.tsx
app/game/components/*
```

## 4. 로비 분석

로비는 `viewMode`를 바꿔가며 세 상태를 가진다.

- `SELECT`
- `QUICK_LIST`
- `CREATE`

즉, 단일 페이지가 게임 포털 역할을 수행한다.

### 빠른 시작

- 즉시 실행형
- 참가자 이름만 추가
- 로컬 상태 기반

### 방 만들기

- Supabase 기반
- 닉네임/비밀번호/한마디 입력
- 친구 초대형

## 5. 상세 방 분석

`useGameRoom`이 이 메뉴의 핵심 엔진이다.

담당 기능:

- 방 정보 로딩
- 참가자 로딩
- 로컬 내 ID/닉네임 복원
- 호스트 판별
- 게임 선택
- 실시간 구독
- 게임 시작/종료
- 참가자 추가/강퇴

즉, 페이지는 조립만 하고 실제 흐름은 훅에 모여 있다.

## 6. 실시간 구조

Supabase channel을 사용해 다음 테이블 변화를 듣는다.

- `game_participants`
- `game_rooms`

따라서 이 메뉴는 황총무 내에서 realtime 성격이 가장 강한 메뉴 중 하나다.

## 7. 지원 게임

코드상 주요 게임:

- 텔레파시
- 광클 대전
- 돌림판
- 사다리 타기

빠른 시작은 로컬형 게임 위주이고, 방 모드는 실시간 룸 상태를 동반한다.

## 8. 사용자 시나리오

### 빠른 시작

1. 게임 선택
2. 참가자 이름 추가
3. 최소 2명 충족
4. 즉시 게임 시작

### 방 만들기

1. 방 생성
2. 친구 참여
3. 호스트가 게임 선택
4. 실시간 상태 동기화
5. 게임 시작/종료

## 9. 강점

- 빠른 시작과 방 모드를 함께 제공한다
- 게임방 로비 UX가 명확하다
- 실시간 구독 구조가 이미 잡혀 있다
- 참가자 관리가 단순하고 진입장벽이 낮다

## 10. 한계

- 인증 강도가 높지 않다
- 게임별 상태 모델이 제각각 커질 수 있다
- 로컬 모드와 서버 모드가 장기적으로 분기될 수 있다

## 11. 개선 제안

- 공통 Game Engine 인터페이스 설계
- 결과 기록/리매치 기능
- 게임별 설정 저장
- 참여자 인증/재입장 UX 강화
- 빠른 시작 결과를 이미지로 공유하는 기능
