# 습관 관리 상세 분석

## 1. 메뉴 목적

습관 관리는 목표 단위의 체크리스트를 매일 기록하고, 그 결과를 월간 달력과 랭킹/댓글과 함께 보여주는 서비스다.

## 2. 라우트 구조

- 생성: `/habit`
- 상세: `/habit/[id]`

## 3. 핵심 파일 구조

```text
app/habit/page.tsx
app/habit/[id]/page.tsx
app/habit/MonthlyTracker.tsx
app/habit/useMonthlyTracker.ts
app/habit/CalendarGrid.tsx
app/habit/TodoList.tsx
app/habit/HabitRanking.tsx
app/habit/CommentSection.tsx
```

## 4. 생성 페이지 분석

생성 페이지는 다음 값을 받는다.

- 목표 제목
- 대표 이모지
- 테마 컬러

즉, 습관 자체를 작은 "방" 또는 "프로젝트"처럼 만든 뒤 상세 추적 화면으로 이동시키는 구조다.

## 5. 상세 구조 분석

상세 페이지는 `MonthlyTracker` 하나가 중심이며, 이 안에서 다시 여러 하위 기능으로 분리된다.

### 좌측

- 달력
- 월/주 전환
- 주말 표시 토글
- 날짜 선택
- 해당 날짜의 체크리스트

### 우측

- 댓글
- 월간 랭킹

즉, 단순 자기관리 도구를 넘어서 "가벼운 소셜/동기부여 요소"를 함께 넣었다.

## 6. 상태와 데이터 흐름

핵심은 `useMonthlyTracker`다.

관리 상태:

- 현재 월
- 선택 날짜
- 주말 표시 여부
- hover 아이템
- 습관 항목 목록
- 월별 로그 요약
- 선택 날짜의 완료 IDs
- raw logs

작동 방식:

1. 목표 항목 조회
2. 현재 월의 로그 조회
3. 선택 일자의 완료 상태 조회
4. 체크 토글 시 DB 반영
5. 월간 로그 재조회

## 7. DB 구조 추정

### `goals`

- 습관 방 메타 정보

### `goal_items`

- 개별 습관 항목

### `goal_logs`

- 특정 날짜의 완료 로그

이 구조는 습관마다 날짜별 상태를 찍는 방식이라 확장성이 좋다.

## 8. 사용자 시나리오

1. 목표 생성
2. 습관 항목 추가
3. 날짜 선택
4. 완료 여부 체크
5. 달력에서 누적 패턴 확인
6. 댓글/랭킹으로 동기 부여

## 9. 기술적 강점

- 로직이 `useMonthlyTracker`로 잘 모여 있음
- 달력/할 일/랭킹/댓글이 명확히 분리됨
- 주말 표시 상태를 localStorage에 저장해 사용자 선호를 기억함

## 10. 한계

- 장기 통계와 회고 화면은 약하다
- 습관 그룹/카테고리 개념이 없다
- 협업형 서비스라기보다는 개인+가벼운 공유 수준에 머문다

## 11. 개선 제안

- 주간/월간 리포트 카드 추가
- streak, best month, relapse 분석
- 습관 카테고리 분류
- 목표별 공유 링크/초대 기능
- 댓글과 랭킹의 가치를 높일 리워드 UX
