# 개인정보처리방침 상세 분석

## 1. 메뉴 목적

개인정보처리방침 고지 페이지. Google AdSense 승인 필수 요건이며, 사이트에서 수집하는 개인정보와 처리 방침을 안내한다.

## 2. 라우트 구조

- 개인정보처리방침: `/privacy`

GlobalHeader 메뉴에는 포함되지 않으며, `/about` 페이지 하단에서 링크로 진입한다.

## 3. 핵심 파일 구조

```text
app/privacy/page.tsx       # 개인정보처리방침 페이지
```

단일 파일로 구성. 별도 컴포넌트 분리 없음.

## 4. 페이지 구조

`PageIntro` + 단일 `StSection` 내에 8개 항목으로 구성된다.

### 항목 구성

1. 수집하는 개인정보 항목
2. 개인정보의 수집 및 이용 목적
3. 개인정보의 보유 및 이용 기간
4. 개인정보의 제3자 제공
5. 광고 및 쿠키 (Google AdSense 관련)
6. 개인정보 보호를 위한 기술적 대책 (SSL/TLS, RLS 등)
7. 개인정보 관리 책임자 (이름, 이메일)
8. 개인정보처리방침 변경

### 데이터 구조

```typescript
const PRIVACY_SECTIONS = [
  { title: string; content: string }
];
```

상수 배열로 관리하며, `white-space: pre-wrap`으로 줄바꿈을 자연스럽게 처리한다.

## 5. 저장 방식

정적 페이지. DB 연동 없음.

## 6. 기술적 강점

- 기존 레이아웃 컴포넌트(StContainer, StWrapper, StSection) 재사용
- 최종 수정일 표시
- `pre-wrap`으로 콘텐츠 내 줄바꿈/리스트 자연스럽게 렌더링

## 7. 구조적 한계

- 콘텐츠가 하드코딩되어 법률 변경 시 코드 수정 필요
- 마크다운 렌더링이 아닌 plain text로 작성되어 링크 등이 클릭 불가

## 8. 개선 제안

- Google 개인정보처리방침 URL을 클릭 가능한 링크로 변환
- 버전 히스토리 추가 (이전 방침 열람 가능)
- 콘텐츠를 MDX로 분리하여 비개발자도 수정 가능하게
