# 업무 캘린더 상세 분석

## 1. 메뉴 목적

업무 캘린더는 프로젝트/서비스별 일정을 보드 단위로 관리하는 내부 운영 도구다.  
황총무 내에서 가장 "업무용 SaaS"에 가까운 메뉴다.

## 2. 라우트 구조

- 로그인: `/login`
- 목록: `/schedule`
- 생성: `/schedule/create`
- 상세 캘린더: `/schedule/[id]`
- 칸반: `/schedule/[id]/kanban`

## 3. 핵심 파일 구조

```text
middleware.ts
app/login/page.tsx
app/schedule/page.tsx
app/schedule/create/page.tsx
app/schedule/[id]/page.tsx
app/schedule/[id]/kanban/page.tsx
services/schedule.ts
app/schedule/components/*
```

## 4. 접근 제어 분석

이 메뉴는 프로젝트에서 유일하게 미들웨어 보호를 받는다.

- `/schedule` 하위 접근 시 쿠키 검사
- `auth_token`이 없으면 `/login`으로 이동

즉, 내부 운영 도구라는 의도가 구조에 반영되어 있다.

## 5. 데이터 모델 분석

`services/schedule.ts`를 기준으로 모델은 3층이다.

- 보드: `schedule_boards`
- 서비스: `schedule_services`
- 태스크: `schedule_tasks`

관계 구조는 아래와 같다.

```text
Board
 └─ Service
     └─ Task
```

이 구조 덕분에 "큰 프로젝트 -> 세부 서비스 -> 일정"으로 나눠 관리할 수 있다.

## 6. 목록 페이지 분석

목록 페이지 역할:

- 보드 목록 조회
- 보드 수정
- 보드 삭제
- 새 보드 생성 진입

카탈로그 페이지로서 동작하며, `ServiceCard`가 카드형 리스트 UI를 만든다.

## 7. 생성 페이지 분석

생성 페이지는 간단한 폼이다.

- 제목
- 설명

입력 후 `createBoard` 호출, 완료 시 바로 상세 보드로 이동한다.

즉, 생성 비용을 낮게 유지하고 상세 편집에서 실제 작업을 하게 하는 설계다.

## 8. 상세 캘린더 페이지 분석

상세 페이지는 좌우 분할 구조다.

### 좌측

- 월간 캘린더
- 서비스별 일정 오버레이
- 태스크 날짜 이동

### 우측

- 서비스 목록
- 태스크 목록
- 숨김 토글
- 전체 상태 편집

핵심 특징:

- `fetchBoardWithData`로 보드와 모든 서비스/태스크를 한 번에 로드
- 서비스 숨김 상태를 프론트에서 `hiddenIds`로 따로 관리
- 포커스 복귀 시 재조회
- 태스크 이동은 optimistic update + 실패 시 재조회

## 9. 칸반 페이지 분석

같은 데이터를 다른 시점으로 재가공한다.

컬럼:

- To-Do
- In Progress
- Done

분류 기준은 완수 여부와 태스크 존재 여부다.

즉, 일정형 관점과 진행형 관점을 동시에 지원한다는 점이 이 메뉴의 강점이다.

## 10. 서비스 레이어 분석

`services/schedule.ts`는 이 메뉴의 핵심 API 레이어다.

역할:

- 보드 CRUD
- 서비스 CRUD
- 태스크 CRUD
- DB row -> 프론트 모델 변환

장점:

- 페이지가 DB 디테일을 직접 몰라도 된다
- date 변환이 service 레이어에 있다

## 11. 사용자 시나리오

1. 로그인
2. 보드 생성 또는 기존 보드 선택
3. 서비스 추가
4. 태스크 추가
5. 일정 캘린더로 배치
6. 칸반에서 진행 상태 확인

## 12. 강점

- 내부 툴 성격이 명확하다
- 보드/서비스/태스크 구조가 실제 업무와 맞다
- 캘린더 뷰와 칸반 뷰를 함께 제공한다
- 미들웨어 보호가 있다

## 13. 한계

- 인증 방식이 단순 쿠키 플래그 수준
- 실시간 협업보다 새로고침/재조회 성격이 강하다
- 페이지가 커질수록 오른쪽 패널 복잡도가 높아질 수 있다

## 14. 개선 제안

- 역할 기반 권한 도입
- 실시간 동기화 또는 presence
- 태스크 의존성 표현
- 담당자 필드 추가
- 히스토리/감사 로그
